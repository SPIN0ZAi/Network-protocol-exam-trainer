🧠 Aprende por capas

Empieza pequeño: primero ARP, luego añade DNS, después DHCP y finalmente la comunicación completa con servidor.

ARP

Se usa cuando PC1 necesita la MAC del siguiente salto local. Si el destino está fuera de la red, se pregunta por el gateway.

IP → MACCapa 2Broadcast

DNS

Solo aparece si hace falta traducir un nombre a IP. Si ya conoces la IP, o está guardada, DNS no sale.

Nombre → IPCachéQuery

DHCP

Aparece cuando el equipo todavía no tiene IP, máscara, gateway o DNS. Entonces pide configuración antes de hablar con nadie.

DiscoverOfferRequestAck

Servidor

Después de resolver todo, viene TCP: SYN, SYN+ACK, ACK, HTTP y cierre FIN/ACK. Si el servidor está fuera, el MAC destino será el gateway.

TCPHTTPGateway
Regla rápida: ARP se usa para llegar al siguiente salto; DNS solo si el nombre necesita resolución; DHCP solo si no hay configuración IP; el servidor entra después de conectar por TCP.
